AsciiDoc (Guidelines)

Table of Contents
  • MUSTS
  • SHOULDS

These are iPipeline guidelines to be used when writing AsciiDoc files. You can also reference the AsciiDoc Syntax Quick Reference you can find online for full documentation.

MUSTS

  • Start all documents with:

    • The Doc Writer

    • An Automatic Table of Contents

    • An explanation of what the document is

SHOULDS

  • It’s useful to include a section for Key Contacts

  • Code examples help bring context to the documents

  • Include a Useful Resources section for links to other material
