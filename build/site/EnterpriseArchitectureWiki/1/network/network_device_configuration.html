Network Device Configuration (Standards)

Table of Contents
  • Firewalls, Switches, Load Balancers

Firewalls, Switches, Load Balancers

Created: 2020-11-24

Updated:

Version: 1.0

  • Administrative Access

    • Interfaces

      • Management access via external [internet facing] interfaces is disabled. device management is performed via dedicated management interfaces that have been specifically configured for management

    • Management protocols

      • Management access is restricted to secure protocols [https, ssh, snmpv3].

    • Cryptography

      • Management interfaces are configured to limit available cryptographic primitives [algorithms, digest, ciphers, etc.] to only the strongest available on the devices. this generally means use of aes for encryption, sha-512 for hashing, tls 1.2, and the longest key lengths available.

    • Login Timeout

      • The login timeout follows the global standard as defined in iPipeline’s IT Ops procedures and standards documentation.

    • Authorized Users

      • All management is performed only by those IT Operations personnel who have been granted a given administrative role authority by the organization to administer and manage these devices/systems.

    • Authentication, Authorization, Accounting

      • Remote aaa is configured using the available protocols on the device [tacacs+, radius, ldap].

    • Two-factor Authentication

      • Administrative access is available only via a VPN connection, which requires two factor authentication.

    • Account Lockout Threshold and Duration

      • The administrative account lockout follows the global standard as defined in iPipeline’s IT Ops procedures and standards documentation.

    • Login Banners

      • Login banners are configured on all devices.

    • snmp

      • Only the required traps are enabled. All other traps are disabled

  • Hostname

    • Hostnames are configured following the global standards

      • Firewalls: fw-(location)-(role)-(number)(letter)

      • Switches: sw-(location)-(role)-(number)(letter)

      • Load balancers: lb-(location)-(role)-(number)(letter)

  • dns

    • dns client settings are configured to used the proper dns servers and search domains.

  • ntp

    • ntp is configured to use the proper internal server[s]

  • Logging

    • Logging is configured at severity level 6 (informational), and sent to the proper syslog server[s]. all console logging is disabled.

  • Traffic Monitoring

    • Traffic monitoring is enabled using supported protocols [netflow, sflow, ect]

  • System Monitoring and Notifications

    • Resource utilization [cpu, memory, filesystem space, temperature, and energy] is monitored by the network monitoring system and notifications are sent when thresholds are exceeded or degraded conditions are detected.

    • Relevant interfaces [connections to firewalls, switches, load balancers, etc.] are monitored, and notifications are sent with interface availability status changes.

  • Unnecessary Features, Interfaces, Protocols, etc

    • All unused features, interfaces, and protocols are completely disabled, and if enabled, will be accompanied by documentation describing the business need that is being satisfied.

  • Interfaces

    • Aggregation

      • All aggregate interface port numbers are configured with the lowest physical port number used

    • Descriptions

      • All critical interfaces are labeled [internet connections, firewall connections, etc.] using the standard convention [remote device hostname]_[remote port]

  • Configuration Backups

    • Configuration backups are configured through the network configuration management system, and daily summary notifications are sent via email.

  • Software Updates

    • Software updates are applied yearly on a rotating schedule depending on device type and location. Off cycle updates are applied in special circumstances (incident resolution, vulnerabilities, ect)
