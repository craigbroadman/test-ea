AWS Account Nomenclature (Specification)

Table of Contents
  • summary
  • base value
    • user accounts
    • component accounts
  • email addresses
    • user accounts
    • component accounts
  • examples

summary

this document describes the nomenclature pattern to be followed for the following aws account settings:

  • account name

  • account alias

  • account email address

the values used for each of the above settings is derived from a base value [${base_value}], as follows:

setting value

account name

${base_value}

account alias

ipipeline-${base_value}

account email address

${base_value}@${email_domain}

base value

the base value format is dependent on the classification of the account. accounts are classified as either user accounts, or component accounts, depending on their purpose:

user accounts

user accounts are used for testing/experimentation/etc., are associated with a single individual [e.g. employee], and managed largely at their discretion, allowing for autonomous manipulation of account objects, services, etc., without the need to coordinate with product owners, teams, etc.

these accounts reside in an independent, aws environment [e.g. payer/root account, control tower management infrastructure, organizations structure, etc.].

structure

email domain: aws.users.iplsys.net
base value format: ${username}-${index}

elements

username

the ipipeline user name of the individual associated with the account

index

an unsigned integer, starting with 1, incrementing as needed for multiple accounts

examples

jdoe-1
jdoe-2

component accounts

component accounts are used for functional operational elements within the organization, and have an identified business owner [often referred to as the "product owner"], with changes or other activities managed by and coordinated with/approved by that owner.

component accounts are further classified as either systems accounts, or services accounts, depending on their purpose:

structure

systems accounts [infrastructure systems, such as directory services, monitoring services, etc.]:

email domain: aws.systems.iplsys.net
base value format: ${component}-${country}-${environment}

services accounts [customer facing products/services, such as business applications]:

email domain: aws.services.iplsys.net
base value format: ${component}-${customer}-${country}-${environment}

elements

component name

the canonical name of the component

customer name

the customer identifier the account is for

accounts that provide services to multiple customers use the reserved shared keyword as the customer name
country

the iso 3166 country in which the audience/consumers of the account’s service[s] are predominately located

countries follow the iso 3166 alpha-3 standard [iso 3166-1] [1] [2] [3]

email addresses

user accounts

for each user, a canonical email address is created [in the form of a distribution list], excluding the index. for each account owned by that user, an alias is created [pointing to the canonical email address] using the base name of each of the accounts.

component accounts

for each product/country [and customer in the case of services accounts], a canonical email address is created [in the form of a distribution list], excluding the environment. for each account within that group, an alias is created [pointing to the canonical email address] using the base name of each of the accounts.

distribution list membership is determined during the planning and design process for a given account deployment.

examples [user accounts]

canonical address : jdoe@aws.users.iplsys.net
alias : jdoe-1@aws.users.iplsys.net
alias : jdoe-2@aws.users.iplsys.net

examples [component accounts]

canonical address : igo-nyl-usa@aws.services.iplsys.net
alias : igo-nyl-usa-dev@aws.services.iplsys.net
alias : igo-nyl-usa-qa@aws.services.iplsys.net
alias : igo-nyl-usa-uat@aws.services.iplsys.net
alias : igo-nyl-usa-prod@aws.services.iplsys.net

examples

user account:

ipipeline username : jdoe
index              : 1
base_value      : jdoe-1
account name    : jdoe-1
account alias   : ipipeline-jdoe-1
email addresses : jdoe-1@aws.users.iplsys.net

services account [shared customer]:

component name : integrations-underwriting
customer name  : shared
country        : gbr
environment    : dev
base_value      : integrations-underwriting-shared-gbr-dev
account name    : integrations-underwriting-shared-gbr-dev
account alias   : ipipeline-integrations-underwriting-shared-gbr-dev
email addresses : integrations-underwriting-shared-gbr-dev@aws.services.iplsys.net

services account [specific customer]:

component name : igo
customer name  : nyl
country        : usa
environment    : prod
base_value      : igo-nyl-usa-prod
account name    : igo-nyl-usa-prod
account alias   : ipipeline-igo-nyl-usa-prod
email addresses : igo-nyl-usa-prod@aws.services.iplsys.net

systems account:

component name : dns
country        : usa
environment    : prod
base_value      : dns-usa-prod
account name    : dns-usa-prod
account alias   : ipipeline-dns-usa-prod
email addresses : dns-usa-prod@aws.systems.iplsys.net
