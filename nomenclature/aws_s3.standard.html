AWS S3 Object Nomenclature (Standards)

Table of Contents
  • summary
  • buckets
    • format
  • objects

summary

this document outlines the nomenclature specifications and standards to be followed when creating s3 objects [buckets, objects, etc.]

to best minimize the potential for conflict, not only among ipipeline infrastrcture, but global aws infrastructure, in addition to the existing constraints imposed by aws [1] [2], the following shall apply:

buckets

due to the above mentioned aws constraints, the following format shall be used for s3 bucket names:

format

${base_value}-${account_number}-${aws_region}

  • words shall be separated from other words with the hyphen [minus sign, dash, etc.] character

objects

1. https://docs.aws.amazon.com/AmazonS3/latest/userguide/bucketnamingrules.html
2. https://docs.aws.amazon.com/en_us/AmazonS3/latest/userguide/object-keys.html
